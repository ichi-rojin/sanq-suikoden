# 歴史生成構造 設計書 v1 ― 歴史を再現せず、歴史が生まれる仕組みを設計する

本書は補足設計思想「本作は歴史を再現するゲームではない」を受け、
既存二書（世界設計書v1・観測系設計書v1）を**構造駆動の原則で監査・再定式化・補強**する。実装は含まない。

---

## 0. 原則の受領と再定式化

### 0.1 二つの設計様式の対比

| | イベント駆動型歴史ゲーム | 本作（構造駆動） |
|---|---|---|
| 歴史の単位 | 固有イベント（赤壁・桃園結義） | 生成構造（決戦構造・結義構造） |
| 発生条件 | 固有名詞への条件分岐 | 状態の勾配（欲望・関係・制度の緊張） |
| 条件を外れたら | 歴史が枯れる | 別の歴史が生まれる |
| シナリオライター | 開発者 | 不在（世界が書く） |
| 物語システムの役割 | 用意した物語の再生 | **生まれた物語の発見** |

### 0.2 禁止事項の形式化（構造駆動三原則）

以下を全設計・全レビューの検収規則とする。

1. **固有名詞禁止則**：ドメイン層のいかなるロジックも、特定の人物名・地名・勢力名・史実イベント名を条件にしてはならない。固有名詞は全て**世界パック（データ）**にのみ存在する（§2）。
2. **台本禁止則**：「AとBが同勢力で、Cが生存していたらイベントXを発生させる」型の条件→結末ルールを禁止する。許されるのは**状態の勾配が行動候補の効用を変える**規則のみ。結末は常に武将の意思決定と偶然の合成でなければならない。
3. **一回性禁止則**：一度しか起きられない仕掛けを禁止する。全ての構造は、条件となる勾配が再び満ちれば**何度でも・誰にでも**発生し得ること。「桃園結義のような集団」は百年に何度も、違う顔ぶれで結ばれてよい。

### 0.3 再定式化された中心の問い

v1の「プレイヤーを削除しても面白いか」に、第二の問いを併置する。

> **「これは歴史を再現しているのか、歴史が生まれる仕組みを設計しているのか。」**

運用形：設計要素を見たとき「この要素は固有名詞を消しても成立するか？別の世界（三國志・SF）に置いても意味を持つか？」——NOなら世界パック行き、それでもNOなら棄却。

## 1. 既存設計の監査

構造駆動三原則で二書を監査した結果。

### 1.1 適合している要素（原則の裏付けを得た）

- **因果保存原則**（v1§1.2）：構造駆動の技術的基盤そのもの。台本がないからこそ、因果リンクだけが物語の背骨になれる。
- **誤射連鎖の設計**（v1§14.2）：「誤射イベントを実装する」のではなく「外れ矢の個別解決＋評価モデル＋想起トリガー」という部品の合成で連鎖が*生まれる*設計になっており、補足思想の誤射例と完全に一致する。
- **水滸伝的定型の創発**（v1§9.2）：野猪林を部品分解し「定型をスクリプトせず、似た構造の別の物語が無数に生まれる」と明記済み。本書はこの一節を全システムへ一般化する作業である。
- **物語パッケージ**（v1§10）：抽出・編纂であり生成ではない。「生まれた物語を発見するシステム」の定義に既に適合。
- **勢力ライフサイクル**（v1§8.3）：一回性がなく、全武将が何周でも興亡できる。

### 1.2 違反の芽（要・再定式化）

| 該当箇所 | 問題 | 処置 |
|---|---|---|
| 招安（v1§8.4） | 「水滸伝固有の名イベント」として設計されている | **取込構造**（§3-5）へ一般化。朝廷×山塞という固有形は世界パックのパラメータ |
| 官府・朝廷（v1§4.3） | 「朝廷」という固有制度がドメインに埋まりかけている | **制度（Institution）**という汎用集約を新設し、朝廷はその一インスタンス（§2.3） |
| 攪拌装置の「遼・金の侵攻」（v1§4.4） | 史実由来の固定シナリオに読める | 攪拌は**外圧・災害・継承危機**という型のみエンジンが持ち、具体（遼・金・洪水）は世界パックの攪拌テーブル |
| 価値観軸「義侠」「面子」（v1§3.2） | 武侠的語彙がドメイン層に混入 | 軸は抽象名（利他・忠誠・野心・獲得・攻撃性・慎重・体面・愛着）で持ち、**表示名と分布**を世界パックが与える |
| 技「妖術」「火計」（v1§7.3） | 技名がドメインに固定 | 技効果プリミティブ（§2.4）＋世界パックでの合成へ分解 |
| 渾名・題テンプレ（v1§10.3） | 「豹子頭」等の語彙 | もとよりテンプレ設計だが、語彙テーブルを世界パック所属と明記 |

**重要な確認**：これらの違反はいずれも「データがロジックに漏れている」型であり、構造そのものの欠陥ではない。監査の結論は*再設計不要・再配置で足りる*。ただし再配置を可能にする器＝世界パックの正式設計が必要（§2）。

## 2. エンジン／世界パック分離アーキテクチャ

### 2.1 分離の定義

```
歴史生成エンジン（普遍）          世界パック（水滸伝は最初の一つ）
──────────────────          ──────────────────────
武将（価値観・欲求・記憶）        価値観軸の表示名・文化的分布
関係・台帳・感情評価              人名/地名/渾名/称号の語彙と生成規則
勢力ライフサイクル・党・放浪      地理グラフの実データ・初期配置
制度（Institution）機構           制度の実体（朝廷・科挙・招安制度）
技効果プリミティブ                技の定義（火計=延焼+恐慌 など）
戦闘解決器・遭遇機構              地形種・遭遇テーブルの重み
攪拌機構（外圧/災害/継承）        攪拌テーブル（遼侵攻・黄河氾濫…）
因果グラフ・史官パイプライン      題テンプレ・台詞テンプレ・史官の文体
観測系（なぜ？/年表/書架…）      観測UIの用語・意匠
```

判別規則：**「if文になるものはエンジン、名詞になるものはパック」**。
エンジンのソースを全文検索して固有名詞が一つでも出たら違反（§5.1 固有名詞監査）。

### 2.2 世界パックのスキーマ（概念）

```
WorldPack =
  地理定義（ノード・エッジ・地形種）
  初期人物群（素質・価値観分布からの生成規則 or 明示リスト=108星）
  制度定義群（§2.3）
  技定義群（プリミティブの合成レシピ）
  大義テンプレート群（§4.2）
  攪拌テーブル（型×頻度×強度）
  語彙パック（人名/渾名/題/台詞/史官文体）
  時代パラメータ（技術水準・移動速度・寿命分布）
```

Phase1で実装するパックは水滸伝のみ。ただし**スキーマは初日から存在させる**。分離は後から入れられない（固有名詞は放置すると全層に根を張る）。

### 2.3 制度（Institution）——新設集約

監査で最大の欠落と判明した概念。勢力（Faction）が「武将の欲望の器」であるのに対し、制度は**個人を超えて存続する規則の器**である。

```
制度 = { 管轄範囲, 規則群（税・登用・処罰・叙勲）, 執行者ポスト, 腐敗度, 正統性 }
```

- 朝廷は「全土管轄・処罰規則・高腐敗」の一制度。科挙・招安・保甲も制度。
- **制度は武将にポストを与え、ポストは行動候補を変える**（護送役は流刑者を運ぶ。腐敗した処罰規則は冤罪を生む）。
- 水滸伝の主旋律「体制が好漢を野に追う」は、`腐敗度の高い処罰制度 × 面子/血気の高い武将` という**勾配の恒常供給**として実装される。三國志世界なら「衰えた漢王朝」、SF世界なら「腐敗した植民地行政」に、同じ機構が差し替えなしで働く。
- 制度は勢力に接収され得る（王朝簒奪）、形骸化し得る（正統性低下）、新設され得る（新王朝の科挙）。**制度自体が興亡する**ことで、百年スケールの「時代の変化」が構造的に生まれる。

### 2.4 技効果プリミティブ

技をエンジン側の効果部品に分解する：

`投射（逸れ判定つき） / 面制圧 / 延焼・汚染（伝播） / 地形改変 / 恐慌・士気操作 / AI目標上書き / 隠密・偽装 / 移動強制`

世界パックが合成する：火計＝延焼＋恐慌、妖術＝面制圧＋恐慌＋目撃者評価タグ「超常」、（SFパックなら）軌道爆撃＝面制圧＋地形改変。**誤射・延焼・波及という物語的性質はプリミティブ側に宿る**ため、どの世界でも戦闘はイベント生成器であり続ける。

## 3. 生成構造カタログ ― 「〜のような出来事」の解剖

本章が本書の核である。史実の名場面を**役割・勾配・部品・出力**に解剖し、普遍構造として登録する。

**カタログの地位に関する最重要注意**：
構造カタログは**実装対象ではない**。「結義構造システム」をコードとして作った瞬間、それは一段抽象化しただけのイベントスクリプトに堕ちる（台本禁止則違反）。
カタログの正しい地位は二つ：
1. **創発の検証目標**——プリミティブ群を回したとき、これらの構造が*自然発生するか*を測る受け入れテスト（v1§18-3の基準シナリオ5種の正式な上位概念）。
2. **ギャップ分析の物差し**——構造が発生し得ないなら、欠けているプリミティブを特定する（§4）。

### 構造一覧

各構造：**勾配**（何が張り詰めると起きるか）／**部品**（どのプリミティブが駆動するか）／**出力**（世界に何を残すか）／**史実の例**（あくまで一インスタンス）。

**3-1. 追放構造**（〜林冲の流罪、水滸伝の主旋律）
- 勾配：腐敗制度の執行者 × 有能で面子/正義感の高い所属者 × 讒言者の私怨
- 部品：制度（処罰規則）・台帳（怨恨）・工作行動（讒言）・護送（移動イベント）
- 出力：在野の英雄＋未清算の怨恨＋制度への敵意。**放浪システムへの人材供給源**
- 一般性：組織が才能を追放する構造は宮廷・軍・企業・教団のどれでも成立

**3-2. 結義構造**（〜桃園結義）
- 勾配：価値観の近い複数者 × 共有された危機or大義 × 相互の恩義台帳
- 部品：遭遇・共闘・大義（§4.2）・結義ラベル（v1§5.3）
- 出力：党の中核、以後の全行動を拘束する紐帯、後の「馳せ参じ構造」の前提

**3-3. 三角工作構造**（〜美女連環の計）
- 勾配：強者Aと腹心Bの紐帯 × それを脅威とみなす第三者C × AとBが同一対象へ向ける欲望
- 部品：**関係を対象とする工作行動**（離間・讒言・貢物・縁組の周旋）＋嫉妬の評価規則
- 出力：紐帯の反転（信頼→怨恨）、勢力中枢の自壊
- 注記：これは「計略イベント」ではない。知略型武将が*他人の関係グラフを読み、そこに介入する行動群*を持てば、C役の武将が自分の効用計算でこれを発明する

**3-4. 決戦構造**（〜赤壁）
- 勾配：一強の急拡大 × 残存勢力の存亡恐怖 × 地形的隘路（渡河・水道・関所）
- 部品：勢力間外交（恐怖駆動の同盟）・地形プリミティブ・技の大規模副作用（延焼）
- 出力：拡大の頓挫、時代区分の転換点、大量の武勇伝と怨恨
- 一般性：「弱者連合が隘路で巨人を止める」は世界を問わない

**3-5. 取込構造**（〜招安）
- 勾配：制度の外にある武力 × 制度側の討伐コスト過大 × 外部勢力内の価値観分裂（体制帰属欲vs反体制）
- 部品：制度（叙勲規則）・外交・勢力内派閥（関係クラスタ）
- 出力：吸収・分裂・使い潰し（腐敗制度下では取込後に追放構造が再発動し、悲劇が構造的に連鎖する）

**3-6. 復讐清算構造**（〜逸れ矢の十年）
- 勾配：未清算の怨恨台帳 × 力関係の逆転（弱者だった側の台頭）
- 部品：台帳・想起トリガー・遭遇の劇的バイアス
- 出力：清算イベント（決闘・暗殺・戦争進言）、新たな怨恨（復讐の連鎖）

**3-7. 馳せ参じ構造**（〜燕青救出のような救出劇）
- 勾配：重い恩義/結義ラベル × 対象の危機情報の到達
- 部品：台帳（恩義）・噂（情報伝達）・移動・戦闘介入
- 出力：救出or殉死、紐帯の強化、救出先勢力との新たな敵対

**3-8. 継承動乱構造**（〜あらゆる御家騒動）
- 勾配：当主の老衰/急死 × 複数の正統性主張者 × 重臣間の派閥
- 部品：勢力の当主継承規則・派閥（関係クラスタ）・野心
- 出力：分裂・粛清・出奔（追放構造への接続）

**3-9. 蜂起構造**（〜方臘の乱のような民衆反乱）
- 勾配：重税/災害による民心崩壊 × 大義を掲げる野心/義侠型武将 × 制度の執行力低下
- 部品：群衆集計値・大義・制度の正統性
- 出力：新勢力の誕生、制度の動揺、討伐戦役

### カタログの読み方

九つの構造は**互いに接続してこそ歴史になる**。
追放（3-1）が放浪者を生み、結義（3-2）が党を作り、蜂起（3-9）や取込（3-5）で時代の主役へ、継承動乱（3-8）と三角工作（3-3）が内から崩し、復讐清算（3-6）が次の火種を残す——この循環図こそが「歴史が生まれる仕組み」の設計図であり、水滸伝も三國志も、この循環の上の一つの軌道に過ぎない。

## 4. ギャップ分析 ― カタログが要求する未設計プリミティブ

カタログを既存設計（v1）と突き合わせた結果、**三つの欠落**が判明した。これが本書の実質的な設計追加である。

### 4.1 工作行動（Scheme Actions）——最大の欠落

v1の武将行動は「移動・会話・取引・戦闘・結義・離反」等、*自分と相手の直接関係*に閉じている。三角工作構造（3-3）と追放構造（3-1）の讒言は、**第三者間の関係を対象とする行動**を要求する。

追加する行動族：
- **讒言**：制度執行者/主君へ、標的の偽情報を吹き込む（噂システムの意図的発信。歪み機構と同一基盤）
- **離間**：二者の間に誤解の種を撒く（偽の噂を双方へ）
- **周旋**：二者を引き合わせる/縁組を仲介する（正の工作。恩義を売る手段でもある）
- **買収**：金銭→忠誠の購入（貪欲値が受容度を決める）

効用計算上、工作は知略が高く武勇が低い武将ほど選好する（能力に応じた手段選択）。これにより**文官型武将に固有の物語**（今まで戦闘でしか輝けなかった層）が生まれる。副次効果として大きい。
工作は失敗・露見し得る。露見は強烈な怨恨と悪名を生む——工作それ自体が新たな物語の火種になる設計。

### 4.2 大義（Cause）——結義・蜂起・正統性の共通基盤

「替天行道」「漢室復興」は固有名詞だが、**旗印を掲げ、価値観の合う者が集う**構造は普遍である。

```
大義 = { 掲げる価値観プロファイル, 敵対対象（制度/勢力/状態）, 掲げた者, 正統性 }
```

- 武将は自分の価値観と大義の適合度で、参集/離反/冷笑を選ぶ。
- 勢力・党・蜂起は大義を持てる（持たない略奪集団も合法）。
- **大義と現実の乖離**（替天行道を掲げつつ略奪する）は、義侠型構成員の失望→離反を駆動する。理想と現実の緊張という文学的主題が、適合度の再評価という機構で表現できる。
- 世界パックは大義テンプレート（語彙と価値観プロファイル）を供給する。

### 4.3 縁組・血縁——継承と同盟の基盤

継承動乱構造（3-8）は血縁と正統性を要求する。v1の世代交代（§4.2）を拡張：
- 婚姻（恋愛からも周旋からも成立）、親子、養子、師弟を関係ラベルとして正式化。
- 継承規則は制度/勢力の属性（長子・実力・指名）。規則と実態のズレが動乱の勾配になる。

## 5. 検証設計 ― 「再現していないこと」をどう保証するか

### 5.1 固有名詞監査（静的検査）

エンジン層（core/chronicle/observatory）の全ソースに対する語彙検査を CI 相当の検収に組み込む。世界パックの語彙テーブルに載る全固有名詞が、エンジン層に一つでも現れたら違反。**構造駆動三原則のうち固有名詞禁止則は機械検査可能**であり、必ず機械化する。

### 5.2 創発検証（動的検査）——カタログを試験仕様書として使う

- 各構造について**検出器**（イベントログ上のパターン照合クエリ）を書く。検出器は observatory の因果クエリ基盤で実装できる（読取専用なので台本禁止則に抵触しない）。
- 100年×複数シードの実行で、九構造それぞれの発生回数・関与人物の多様性（毎回同じ顔ぶれなら固定化の兆候）・構造間の接続率（追放→結義の連鎖率など）を計測。
- **v1 DoDの「基準シナリオ5種」を本カタログ準拠に改訂**：誤射→離反（3-6の一形）、冤罪→入山（3-1→3-2）、恩義→馳せ参じ（3-7）、内紛→分裂（3-8）、滅亡→再興（勢力ライフサイクル）に加え、Phase1.5で三角工作（3-3）・取込（3-5）を追加検収。

### 5.3 第二世界試験（エンジン性の最終証明）

「世界設定を差し替えるだけで異なる歴史が生成される」は、差し替えてみるまで真実にならない。
- Phase1.5に**ミニマル第二パック**を置く：地理20ノード・生成人名・制度1種・技3種の抽象世界（三國志風で可）。制作規模は語彙とデータのみで数日規模を目標。
- 合格基準：エンジン無改修で完走し、九構造の検出器が第二世界でも発火すること。
- この試験は製品ではなく**アーキテクチャの結合検査**である。ただし合格した瞬間、本プロジェクトの自己定義（英雄譚生成エンジン）が検証済みの事実に変わる。

## 6. 批判的レビュー

### 6-1. 抽象化税
汎用化は常にコストを持つ。制度・大義・工作・プリミティブ合成は、水滸伝直書きに比べ設計・実装とも重い。**過剰一般化（まだ存在しない第二世界のための設計）は第一世界の完成を殺す**古典的リスク。
→ 応答：本書の分離は「実装の抽象化」ではなく「データの再配置」が主であり（§1.2監査の結論）、追加実装は工作・大義・制度の三点に限定されている。三点はいずれも水滸伝世界*そのもの*が要求する（讒言なき水滸伝、替天行道なき梁山泊、腐敗官府なき追放は成立しない）。**第二世界のためではなく第一世界のために必要**——これが過剰一般化でない根拠である。

### 6-2. 汎用エンジンは無味になる危険
普遍構造だけの世界は「どの世界も同じ味」になり得る。歴史の味は固有の語彙・地理・文化的偏りに宿る。
→ 応答：味は世界パックの責務であり、パック制作は語彙テーブル埋めではなく**文化の記述**（価値観分布の偏り、大義テンプレの思想、攪拌テーブルの気候）である。水滸伝パックの品質基準として「体制起因の没落数／義による結合数」メトリクス（v1§17-3）を維持する。エンジンが普遍でも、計測で世界の個性を守る。

### 6-3. 構造カタログ自体が「上位の台本」になる危険
検証目標と言いつつ、九構造の発生率をチューニングし始めれば、それは間接的なシナリオ執筆ではないか。
→ 応答：半分正しい。調整が許されるのは**勾配の供給量**（制度腐敗度・攪拌頻度＝環境側）のみで、構造の発生そのものを狙い撃つパラメータ（「結義発生率」等）を作ることは禁止する。v1§16-5の「介入は環境のみ、武将の意思には触れない」の線引きを、チューニング規律として明文化する。

## 7. 自己レビューと改善（3回）

### 自己レビュー第1回：価値観軸の抽象化は正しかったか
§1.2で軸を抽象名に変えたが、「義侠」を「利他」に均すと水滸伝的な*任侠の美学*（見知らぬ他人のために体制と敵対する非合理）が薄まる懸念がある。
→ 改善：軸は抽象、**評価規則の重み付けを世界パックが上書き可能**とする（例：水滸パックでは「目撃した不義」への利他反応の重みを高く設定）。文化とは価値観軸の名前ではなく、**同じ軸の発火のしかたの偏り**である、と定式化する。

### 自己レビュー第2回：工作行動と噂システムの依存関係が矛盾
§4.1の讒言・離間は噂の歪み機構を前提とするが、v1§16-2で歪みはPhase1.5送りにした。三角工作もPhase1.5検収（§5.2）なので日程矛盾はないが、**追放構造（Phase1必須）も讒言を要求する**ため穴がある。
→ 改善：Phase1の讒言は歪み一般機構を待たず、**「執行者への直訴」という単純形**（偽情報を制度執行者一人に注入する行動）で先行実装する設計に変更。一般の噂歪みネットワークとは独立に成立し、Phase1.5で歪み機構へ統合する。段階計画の整合を回復。

### 自己レビュー第3回：マイルストーンへの反映漏れ
本書の追加要素（制度・大義・工作・固有名詞監査・第二世界試験）がM1〜M5に載っていない。
→ 改訂：
- **M0（新設）**：世界パックスキーマ確定＋固有名詞監査の機械化（以後全Mの検収に常時掛ける）
- **M1**：＋制度の骨格（処罰規則と冤罪発生のみ）
- **M2**：＋讒言（直訴形）——追放構造の検出器で検収
- **M3**：＋大義（結義・蜂起に接続）＋継承規則
- **M4**：（変更なし：史官・書架）
- **M5**：＋構造カタログ九種の発生統計をDoD計測に組込み
- **Phase1.5**：噂歪み統合・三角工作・取込・複数史観・**第二世界試験**

### 自己レビュー第4回（追加）：本書の位置づけの明確化
三冊の設計書の関係を定義しておく。
- 世界設計書v1 … **何が世界を動かすか**（生成）
- 観測系設計書v1 … **どう面白さに到達するか**（発見）
- 本書 … **なぜそれが水滸伝の再現ではなく歴史生成なのか**（原理と分離）
以後の全設計判断は、v1の問い「プレイヤーを削除しても面白いか」と本書の問い「再現か、生成か」の**二重検問**を通過すること。

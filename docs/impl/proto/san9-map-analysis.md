# 三國志IX 中国全土マップ分析（Prototype Upgrade 前提分析）

> 出自: Producer提供のSAN9全土マップ画像（3200×1650、アイソメトリック）の目視計測＋公開情報。
> 目的: 画像の再現ではなく、論理マップ（真上視点グリッド）への変換仕様と
> 「眺めて面白い」設計原則の抽出。裁定R-16の添付分析。

## 1. グリッド構造（画像推定 → Web調査で更新）

**画像からの初期推定**:

| 項目 | 計測・推定 |
|---|---|
| タイル描画ピッチ | 幅≒16px・高さ≒8px（2:1アイソメ。海岸線の階段状ジャギの1段から計測） |
| 菱形の全幅 | ≒3,150px → (W+H)×8 ≒ 3,150 → **W+H ≒ 394〜400** |
| 頂点の東西偏差 | 上頂点 x≒1,500 → H×8≒1,470 → W≒216 × H≒184 と推定 |

**Web調査による更新（2026-07-11）**: 改造ツールSan9MEの解析情報により、内部は
**ヘックスではなくスクエア（方形マス）**で、**200×200グリッド（square番号=200y+x）**
という記述を確認（原典ページ閉鎖のため推測扱いだが、改造スレの座標観測値
X≦≒172・Y≦≒140とも矛盾しない。画像推定のW+H≒400とも整合的）。
**本Prototypeは 200×200 を採用**（`world.data.ts` の GRID_W/GRID_H。執筆座標は自動再スケール）。

## 2. 拠点の計測

- 大型城郭（灰青屋根）: ≒14 — 州都級。平野の河川沿いに集中
- 中型城郭（土壁）: ≒30 — 郡県級
- 赤壁の砦: ≒16 — 西南・南方（異民族・辺境拠点）
- 小型旗付き小屋: ≒40 — **港・関**。河川の渡河点、山地の隘路、海岸に規則的に分布

**Web調査による確定値**: 都市50（通常45＋異民族4＋隠し1）、**港34・関8・砦1、計185地域**
（攻略サイトの都市データ一覧・中国解析記事による）。
ターン構造は公式マニュアルで **1ターン=1旬=10日、1年=36進行フェイズ** を確認。
移動速度は陣形の機動力（長蛇20〜方円8）で決まり、**隣接都市まで最速陣形で概ね1.5〜4旬**。
本Prototypeの「1辺=1ヶ月（3旬）」はこの距離感の範囲内にある。
兵法発動・一騎討ちは「拡大画面への切替」で演出される——本Prototypeでは
マップ上の合戦盤面＋発動ポップとして翻案。常時UIの**レーダーマップ**（勢力色の
全土ミニマップ、クリックでジャンプ）もViewerへ実装した。

**一都市の平均支配領域**: 39,744セル ÷ 50 ≒ **795セル（半径≒16セル）**。
**都市間距離**: 隣接都市間 ≒18〜30セル。行軍数旬〜1ヶ月の距離感。

## 3. 地形構造の分析

- **河川が主役**: 黄河・長江が画面を大きく横断し、渡河点（港）でしか渡れない。
  河川は「壁」であり、港が「扉」。長江沿いの港列が南北戦争の全導線を規定する
- **山岳は回廊を作る**: 太行山脈・秦嶺・蜀の山塊が平野を区切り、
  関（潼関・剣閣型の隘路）だけが通り道。山塊は迂回不能な戦略的固定壁
- **森林は中間地帯**: 平野の縁に点在し、視覚的な「地方の境目」を作る
- **ボトルネック**: 渡河港・関・半島の付け根。ここで軍勢が自然に正面衝突する
- **防衛ライン**: 河川（黄河=北の内戦線、長江=南北戦線）＋関の連なりが
  「面ではなく線」の防衛を成立させる
- **侵攻ルート**: 中原→四方への放射街道と、運河・大河沿いの縦貫路の二層

## 4. 画面・テンポの分析

- **カメラ倍率**: 全土俯瞰では都市アイコンと軍旗だけが読め、寄れば部隊の点が読める2段階
- **情報密度**: 常時数十の動体（部隊・輸送・使者）が地図上を這う。
  「どこかで必ず何かが動いている」状態が維持される
- **ゲームテンポ**: 戦略決定（旬・月次）と、連続的な行軍アニメの2層。
  意思決定の粒度は粗く、画面の変化は細かい

## 5. なぜ三國志IXは眺めていて面白いのか（移植する設計原則）

1. **単一シームレスマップ** — 全戦争が同じ空間で起きるため、遠くの戦火も「同じ世界の出来事」として目に入る
2. **地理が戦略を語る** — 河・山・関が導線を絞り、会戦が毎回「名のある場所」で反復する。場所に記憶が宿る
3. **常時可動体** — 行軍・輸送が画面のどこかで常に動き、世界の鼓動になる
4. **プレイヤー不在でも進む進行フェイズ** — 命令後は世界が勝手に動き、観察だけが残る
5. **情報の遠近法** — 俯瞰では勢力色と旗、接近では個の名前。ズームが「国家の目」と「個人の目」を切り替える

## 7. 追加Web調査による裏取り（2026-07-11、Producer指示による優先調査）

レビュー・攻略Wiki・Steamレビュー等の言説調査で確認できた事実と、本Prototypeへの反映:

| 確認事項 | 出典の要旨 | 反映 |
|---|---|---|
| 1ターン=10日（1ヶ月=3ターン、年36ターン） | GAME Watchレビュー | 月次tickの中を連続的な行軍アニメで埋める（旬相当の細かさは演出側で表現） |
| 「武将が勝手に動き戦闘するのを見るだけで楽しい」「監督気分」 | シリーズ横断レビュー | 介入なし進行＋追跡カメラを既定に |
| 一枚マップゆえに多正面同時戦争が起きる | GAME Watch・個人ブログ | 合戦はモーダルでなくマップ上で並行再生（世界は止めない） |
| 兵法のランダム発動（奮迅・斉射など）が「癖になる」 | Steamレビュー | 技発動を戦場ポップ（突撃/斉射/火計…）として常時可視化 |
| 進行フェイズの速度調整不可が最大級の不満 | Steamガイド | 速度×0.5〜×8と一時停止を最初から装備 |
| マウスドラッグ/ホイール非対応・低解像度が不評 | Steamレビュー | 自由ドラッグ・ホイールズーム・追跡カメラを実装 |
| 中盤以降の作業化・内政の薄さが弱点 | 複数レビュー | 本作は「眺める」特化のため対象外（M1以降の設計判断材料として記録） |

内部グリッドの正確なマス数は一次情報が見つかっておらず（追加調査継続中）、
本Prototypeは§1の画像計測による推定 **216×184** を採用した。確定値が得られた場合は
`packages/slice/data/world.data.ts` の `GRID_W/GRID_H` の差し替えのみで追従できる。

## 8. 第2次Upgrade（裁定R-17）: 世界モデルそのもののTile化

R-16は「見た目のタイル化」に留まり、シム本体は都市ノード＋街道エッジのグラフのままだった。
Producer発注（2026-07-11 第2次）により、**世界モデルそのものを200×200のTileMapへ刷新**した。

| 対象 | R-16まで | R-17 |
|---|---|---|
| 空間 | 拠点グラフ（71ノード・86辺） | 200×200=40,000セル（`src/grid.ts`）。河=壁・渡し場=扉・関=峠の回廊を地形として生成 |
| tick | 1ヶ月 | **1日**（人生の節目は月次、勢力戦略は四半期） |
| 移動 | 辺のTween（1辺=1ヶ月） | タイルを一歩ずつ（A*経路・地形別移動コスト。街道0.34/平地0.6/森1.15…）。隣接都市まで実測10〜15日≒SAN9の1.5〜4旬 |
| 戦闘 | 別盤面13×13のリプレイ | **世界そのものが戦場**（`src/field.ts`）。交戦は世界タイル上で日次進行、第三勢力の途中乱入（war.join）・離脱・追撃・攻城（城壁・城門・破城）を含む |
| 技 | リプレイ盤面内で完結 | 世界へ作用: 火計は実地形を延焼し森林火災・都市延焼に至る（焼け跡は約2年残る）、崖崩れは瓦礫タイルで道を塞ぐ（約半年）、斉射は投射物として持続し旅人も巻き込む、伏兵は森タイルに潜伏 |
| 小窓 | 合戦リプレイ再生 | 基本戦闘から除外。一騎討ち・義盟・処刑・奪還・酒宴・冤罪・帰順の**人物ドラマ演出専用**（`viewer/drama-view.ts`）。上演中も世界は止まらない |
| 都市 | ノード | 敷地タイル＋城壁の環＋四門。攻城戦は門を破って本丸へ雪崩れ込む実過程 |
| 護送・放浪 | 辺単位の瞬間移動 | タイル旅程。友は護送路へ先回りし、林の難所で奪還の賭けが日々起きる。街道では追い剥ぎ・擦れ違いの遭遇が発生 |

旧グラフ世界のPrototypeは保全ブランチ **`prototype/graph-world-v1`**（8dfdbfe）に恒久保存し、
「何が良かったか／何が足りなかったか」の将来比較に備える。
